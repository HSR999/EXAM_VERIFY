# Two-Minute Demo Script

**0:00 - Problem**

Every year, millions of students enter high-stakes exams, but identity checks still depend on an invigilator comparing a card and a face in seconds.

**0:12 - Home**

ExamVerify connects official digital documents, a live face signal, fraud rules, and a complete audit trail.

**0:20 - Verified candidate**

Open Student Verification and choose `JEE25BPL0042`. Point out the DigiLocker-compatible payload, verified issuer, and live API log. Start the face match and show the 94.2% approval.

**0:52 - Fraud case**

Choose the next candidate and select `JEE25BPL0103`. The confidence stops at 61.3%, low-confidence and repeated-attempt flags are raised, and the invigilator is alerted.

**1:15 - Dashboard**

Open the dashboard. Show verified, flagged, and pending counts, candidate search, confidence bars, and fraud signals.

**1:32 - Audit trail**

Open Audit. Filter to Critical, point out the append-only events, and export the CSV.

**1:48 - Close**

ExamVerify uses existing infrastructure, ordinary webcams, and a privacy-first workflow to make exam entry faster and more accountable. The current build is an honest sandbox prototype, ready for approved DigiLocker and biometric providers.
